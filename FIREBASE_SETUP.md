# Firebase Setup for HomHom

This guide walks you through setting up Firebase for in-app purchases and HOM balance management.

## Prerequisites

- Google account
- Firebase CLI installed: `npm install -g firebase-tools` ✅ (already done)
- HomHom project cloned locally ✅ (already done)

---

## **Step 1: Create Firebase Project**

### 1.1 Go to Firebase Console
- Open https://console.firebase.google.com
- Click **"Create a project"** or **"Add project"**

### 1.2 Project Details
- **Project name:** `homhom-app`
- **Analytics:** Optional (uncheck if you want to skip)
- Click **"Create project"**

### 1.3 Wait for Project Creation
Takes ~30 seconds. You'll see a "Project is ready" message.

---

## **Step 2: Configure Authentication**

### 2.1 Enable Email/Password Auth
1. Go to **Build → Authentication**
2. Click **"Get started"**
3. Click the **"Email/Password"** provider
4. Toggle **"Enable"** → **"Save"**

### 2.2 (Optional) Enable Anonymous Auth (for testing)
1. In Authentication providers list
2. Click **"Anonymous"**
3. Toggle **"Enable"** → **"Save"**

---

## **Step 3: Create Firestore Database**

### 3.1 Initialize Firestore
1. Go to **Build → Firestore Database**
2. Click **"Create database"**

### 3.2 Firestore Setup Dialog
- **Location:** Choose closest to your users (e.g., `europe-west1`)
- **Security rules:** Select **"Start in test mode"** for now
- Click **"Create"**

### 3.3 Replace Security Rules
1. Go to **Firestore → Rules** tab
2. Delete existing rules
3. Copy the content from `firestore.rules` in your project
4. Paste it in the editor
5. Click **"Publish"**

---

## **Step 4: Deploy Cloud Functions**

### 4.1 Get Service Account Credentials
1. Go to **Project Settings** (⚙️ icon, top right)
2. Click **"Service Accounts"** tab
3. Click **"Generate new private key"**
4. Save the JSON file as `functions/credentials.json` (keep it secret!)

### 4.2 Set Environment Variables
```bash
cd ~/development/flutter_projects/homhom

# Set your OpenAI API key
export OPENAI_API_KEY="your_openai_api_key_here"
export GOOGLE_PACKAGE_NAME="com.homhom.app"  # Update if different
```

### 4.3 Install Dependencies
```bash
cd functions
npm install
cd ..
```

### 4.4 Login to Firebase CLI
```bash
firebase login
```

A browser will open. Sign in with your Google account.

### 4.5 Deploy Cloud Functions
```bash
firebase deploy --only functions

# You should see:
# ✔ Deploy complete!
# ✔ functions deployed successfully
```

### 4.6 Verify Deployment
Check Cloud Functions in Firebase Console:
- **Build → Functions**
- You should see 4 functions:
  - `validatePlayPurchase`
  - `processMeal`
  - `getUserBalance`
  - `setApiKey`

## **Step 5: Set Up Android**

### 5.1 Download Google Services Config
1. Go to Firebase **Project Settings** (⚙️)
2. Click **"Your apps"** → **Android app** (or add one)
3. Enter **Android package name:** `com.homhom.app` (or your actual package)
4. Click **"Register app"**
5. Click **"Download google-services.json"**
6. Place file at: `android/app/google-services.json`

### 5.2 Update Android Build Files

Add to `android/build.gradle.kts`:
```kotlin
buildscript {
  repositories {
    google()
    mavenCentral()
  }
  dependencies {
    classpath("com.google.gms:google-services:4.4.0")
  }
}
```

Add to `android/app/build.gradle.kts`:
```kotlin
plugins {
  id("com.google.gms.google-services")  // Add this
}

dependencies {
  // Firebase BOM (Bill of Materials)
  implementation(platform("com.google.firebase:firebase-bom:32.7.1"))
  implementation("com.google.firebase:firebase-core")
  implementation("com.google.firebase:firebase-auth")
  implementation("com.google.firebase:firebase-firestore")
}
```

## **Step 6: Set Up iOS** (if targeting App Store)

### 6.1 Download iOS Config
1. Firebase **Project Settings** → **Your apps** → **iOS app**
2. Enter **iOS bundle ID:** `com.homhom.app` (or your ID)
3. Click **"Register app"**
4. Click **"Download GoogleService-Info.plist"**
5. Open Xcode: `open ios/Runner.xcworkspace`
6. Drag `GoogleService-Info.plist` into Xcode (check "Copy if needed")

### 6.2 CocoaPods Setup
```bash
cd ios
pod install
cd ..
```

## **Step 7: Update Flutter App**

### 7.1 Get Flutter Packages
```bash
cd ~/development/flutter_projects/homhom
flutter pub get
```

### 7.2 Update main.dart

Modify `lib/main.dart` to initialize Firebase:

```dart
import 'package:firebase_core/firebase_core.dart';
import 'package:provider/provider.dart';
import 'firebase_options.dart';  // Generated by flutterfire configure
import 'providers/firebase_provider.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  
  // Initialize Firebase
  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );
  
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MultiProvider(
      providers: [
        ChangeNotifierProvider(create: (_) => FirebaseProvider()),
      ],
      child: MaterialApp(
        title: 'HomHom',
        theme: ThemeData(
          primarySwatch: Colors.blue,
          useMaterial3: true,
        ),
        home: const HomeScreen(),
      ),
    );
  }
}
```

### 7.3 Generate Firebase Config
```bash
dart pub global activate flutterfire_cli
flutterfire configure
```

This generates `lib/firebase_options.dart`.

## **Step 8: Test Firebase Locally**

### 8.1 Run Emulators
```bash
firebase emulators:start
```

You'll see:
```
✔ Emulator Suite started
  Auth Emulator: http://127.0.0.1:9099
  Firestore Emulator: http://127.0.0.1:8080
  Functions Emulator: http://127.0.0.1:5001
```

### 8.2 Update Firebase Connection (for testing)
In `lib/services/firebase_service.dart`, add after `Firebase.initializeApp()`:

```dart
if (kDebugMode) {
  // Use local emulators for testing
  await FirebaseAuth.instance.useAuthEmulator('localhost', 9099);
  FirebaseFirestore.instance.useFirestoreEmulator('localhost', 8080);
}
```

### 8.3 Run Flutter App (in another terminal)
```bash
flutter run
```

## **Step 10: Configure Google Play Billing (Google Cloud)**

### 10.1 Link Google Cloud Project
1. Firebase **Project Settings** → **General**
2. Scroll to "Google Cloud Console" section
3. Click the Cloud project link

### 10.2 Enable Google Play Android Developer API
1. In Google Cloud Console
2. Search **"Google Play Android Developer API"**
3. Click → **"Enable"**

### 10.3 Create Service Account (for receipt verification)
1. Go to **APIs & Services → Credentials**
2. Click **"Create Credentials"** → **"Service Account"**
3. Fill in details (any name is fine)
4. Click **"Create and Continue"**
5. Skip optional steps, click **"Done"**
6. Click the service account you just created
7. Go to **"Keys"** tab
8. Click **"Add Key"** → **"Create new key"**
9. Choose **"JSON"** → **"Create"**
10. Save as `functions/credentials.json`

### 10.4 Grant Permissions
Back in service account details:
1. Click the email address
2. Click **"Details"** tab (or refresh)
3. You'll see the service account email
4. In Google Play Console → **Settings → Users and permissions**
5. Invite the service account email
6. Give it **"Admin"** role

## **Common Issues**

### Issue: "Package name doesn't match"
- **Fix:** Make sure package name in `AndroidManifest.xml` matches Firebase config

### Issue: "Cloud Functions not deploying"
- **Fix:** Run `npm install` in `functions/` directory first

### Issue: "Firestore says Access Denied"
- **Fix:** Check security rules. Start with test mode, then replace with rules from `firestore.rules`

### Issue: "OpenAI API failing"
- **Fix:** Verify `OPENAI_API_KEY` environment variable is set when deploying functions
